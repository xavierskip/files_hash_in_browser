# 项目说明

这是一个**基于浏览器的文件哈希比对工具**。

## 计划 plan

 - [x] 增加其他种类的hash方式，而是固定的SHA-1。具体的其他hash方式由函数`crypto.subtle.digest`所支持。
   - [x] 除了通过页面UI调整，还可以通过URl的参数来设定。比如`?hash=sha-256`
   - [x] 页面UI调整的同时同步修改URL中的相关参数
- [x] 生成的文件hash结果可以通过点击切换显示模式，默认是紧凑模式（中间没有空格隔开），点击后切换到人类视力友好模式（hash结果中有空格隔开）。
  - [x] 同样的文件 size 栏数据，也要如此，除了展示比特数，最好能通过点击切换展示更人类友好的 kb、mb单位显示。
- [x] 错误处理：用户拖拽文字内容到此页面。 
  - [x] 处理拖放文件包含文件夹的情况。
- [x] 离线使用。(不熟悉，还需多测试、观察)
- [x] 移动浏览器测试、支持。

## 待讨论/问题

- 算法选择目前依靠 URL 查询参数，是否有必要在本地 cookie 中记录？


## 功能

- 将文件拖拽到网页指定区域
- 使用 Web Crypto API 计算每个文件的 SHA1 哈希值
- 在表格中显示文件的哈希值、大小和名称
- 将所有文件的哈希值与**第一个文件的哈希值**进行比对
  - 哈希值相同的显示为**绿色**
  - 哈希值不同的显示为**红色**
- 并行处理多个文件

## 技术栈

- 纯 HTML/CSS/JavaScript（无外部库）
- 使用原生浏览器 API：FileReader、crypto.subtle.digest()、拖放 API
- 单页 Web 应用，部署在 Vercel 上

## 已知限制

- 超过 2GB 的文件可能因浏览器沙箱限制而无法计算哈希值
